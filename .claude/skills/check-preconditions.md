# check-preconditions

알고리즘/기법을 코드에 적용하기 전에 전제조건과 현재 시뮬레이션 환경의 적합성을 검증합니다.

## 언제 사용하나

- 새로운 제어 알고리즘(논문/교과서)을 구현하려 할 때
- Space_SLAM에서 검증된 알고리즘을 ROS 2 노드로 이식하려 할 때
- 다른 도메인(지상, 실내)의 기법을 우주 시나리오에 적용하려 할 때

## Steps

1. 인자로 알고리즘/기법 이름을 받음 (예: `/check-preconditions impedance_control`, `/check-preconditions ESKF`)
2. 해당 알고리즘의 **전제조건**을 나열:
   - 수학적 가정 (선형성, 가우시안, 관측 가능성 등)
   - 데이터 요구사항 (센서 주파수, 정밀도, 토픽 가용성)
   - 환경 가정 (무중력, 강체, Gazebo 물리엔진 특성)
   - ROS 2 요구사항 (필요 토픽, 서비스, 타이밍)
3. 현재 프로젝트 시나리오와 **대조표** 작성:

```
| 전제조건 | 요구값 | 현재 환경 | 판정 |
|----------|--------|----------|------|
| 무중력 | 필수 | gravity=0 (SDF) | PASS |
| IMU 주파수 | >100Hz | Gazebo 1kHz | PASS |
| 실시간성 | <10ms | WSL2 오버헤드 | WARN |
```

4. FAIL 항목이 1개 이상이면:
   - **경고**: "이 알고리즘은 현재 시뮬레이션에서 [FAIL 사유]로 인해 기대 성능을 달성할 수 없습니다"
   - **대안 제시**: 조건을 만족하는 대체 접근법 또는 수정 방향
5. 모두 PASS이면:
   - **승인**: "전제조건 충족. 구현 진행 가능"
   - 주의사항이 있으면 함께 명시

## 주의: 단위 체계 차이

- Space_SLAM: km, km/s (MATLAB)
- orbit_sim: m, m/s (ROS/Gazebo)
- 이식 시 단위 변환 누락이 가장 흔한 실수!
